1. Общее описание проекта

1.1 Цель проекта
BarberGuide — это цифровая платформа, решающая проблему неопределённости клиентов барберских салонов при выборе стрижки. Большинство мужчин приходят к мастеру без чёткого представления о желаемом результате, что снижает качество обслуживания, увеличивает время на консультацию и уменьшает поток клиентов у мастера.
Платформа предоставляет клиентам инструменты для осознанного выбора стрижки, а барберам — канал привлечения новых клиентов.
1.2 Целевая аудитория
Сегмент
Описание
Мужчины 16–40 лет
Основные пользователи: ищут подходящую стрижку, хотят выглядеть актуально
Барберы и мастера
B2B-сегмент: регистрируют профиль, привлекают клиентов через платформу
Барбершопы / салоны
Корпоративный сегмент: добавляют команду мастеров, управляют записями
1.3 Платформы
	•	Веб-сайт (React + Spring Boot) — разрабатывается первым разработчиком
	•	Мобильное приложение (iOS + Android) — разрабатывается вторым разработчиком
	•	Единый бэкенд (Spring Boot REST API) — используется обеими платформами
	•	Единая база данных (PostgreSQL) — общая для веб и мобайла

2. Функциональные требования

2.1 Модуль: Каталог стрижек
2.1.1 Просмотр каталога
	•	Сетка карточек стрижек (grid layout): фото, название, теги
	•	Бесконечная прокрутка или пагинация (по 20 элементов)
	•	Сортировка: по популярности, новизне, алфавиту
	•	Карточка стрижки: галерея фото (несколько ракурсов), описание, теги, рекомендации по уходу, кнопка «Сохранить»
	•	Похожие стрижки в нижней части страницы стрижки
2.1.2 Фильтрация и поиск
Фильтры доступны как на странице каталога, так и как результат прохождения квиза:
	•	По форме лица: овальное, квадратное, прямоугольное, треугольное, круглое, ромбовидное
	•	По типу волос: прямые, волнистые, кудрявые, тонкие, густые, жёсткие
	•	По длине: короткая, средняя, длинная
	•	По стилю: классический, современный, андеркат, фейд, тейпер, помпадур, текстурный
	•	По времени на укладку: до 5 мин, 5–15 мин, 15+ мин
	•	Полнотекстовый поиск по названию и тегам
	•	Сохранение активных фильтров в URL (для шаринга)
2.2 Модуль: Подбор стрижки (Квиз)
Пошаговый мастер из 4–6 вопросов, результатом которого является персонализированная подборка стрижек:
	•	Шаг 1 — Форма лица (с иллюстрациями для понимания)
	•	Шаг 2 — Тип волос (с примерами фото)
	•	Шаг 3 — Желаемая длина
	•	Шаг 4 — Стиль жизни (деловой, спортивный, творческий, casual)
	•	Шаг 5 — Сколько времени готов тратить на укладку
	•	Шаг 6 (опционально) — Какие стрижки нравились раньше
По итогам квиза отображается топ-5 рекомендованных стрижек с процентом совпадения.
2.3 Модуль: Тренды
	•	Отдельная страница с актуальными трендами сезона
	•	Редакционные подборки (например: «Топ-10 стрижек лета 2025»)
	•	Фильтр по году/сезону
	•	Возможность добавить тренд в сохранённые
2.4 Модуль: Карточка для мастера
Ключевая функция, решающая проблему коммуникации с барбером:
	•	После выбора стрижки генерируется карточка
	•	На карточке: фото стрижки, официальное название, профессиональная терминология (fade, taper, texture, длина в сантиметрах)
	•	Кнопка «Скачать карточку» (PNG/PDF)
	•	Кнопка «Поделиться» (ссылка или через мессенджеры)
	•	QR-код, ведущий на страницу стрижки — барбер может отсканировать
2.5 Модуль: Поиск барбера
	•	Карта с отображением барберов поблизости (Google Maps / Yandex Maps API)
	•	Фильтр: рейтинг, цена, специализация (например, владеет fade)
	•	Страница профиля барбера: фото работ, описание, цены, рабочие часы
	•	Кнопка «Записаться» → переход к онлайн-записи
	•	Отзывы и рейтинг клиентов
2.6 Модуль: Онлайн-запись (v2)
	•	Выбор мастера, даты и времени
	•	Прикрепление выбранной стрижки к записи
	•	SMS / email подтверждение
	•	Напоминание за 24 часа
	•	История записей в личном кабинете
2.7 Модуль: Личный кабинет пользователя
	•	Регистрация / вход: email + пароль, Google OAuth, Apple ID (для мобайла)
	•	Сохранённые стрижки (вишлист)
	•	Профиль: имя, фото, форма лица, тип волос (заполняется один раз)
	•	История записей и посещённых барберов
	•	Напоминание «Пора подстричься» через N недель после последней записи
2.8 Модуль: Кабинет барбера (B2B)
	•	Регистрация как мастер с верификацией
	•	Профиль: имя, фото, специализации, адрес, цены, рабочее время
	•	Портфолио: загрузка фото работ с тегами стрижек
	•	Управление расписанием и записями
	•	Аналитика: просмотры профиля, количество записей, популярные стрижки

3. Нефункциональные требования

3.1 Производительность
	•	Время загрузки главной страницы: не более 2 секунд
	•	Время ответа API: не более 300 мс для большинства запросов
	•	Поддержка одновременных подключений: не менее 1000 пользователей
	•	Оценка Lighthouse Performance: не менее 85/100
3.2 Безопасность
	•	Авторизация через JWT токены (Access + Refresh)
	•	HTTPS обязательно на всех эндпоинтах
	•	Хэширование паролей: BCrypt
	•	Защита от SQL-инъекций через JPA / параметризованные запросы
	•	Rate limiting на API: не более 100 запросов в минуту с одного IP
	•	CORS настроен под конкретные домены фронтенда
	•	Валидация всех входящих данных на уровне бэкенда
3.3 Масштабируемость
	•	Бэкенд упакован в Docker-контейнер
	•	Возможность горизонтального масштабирования (stateless-архитектура)
	•	CDN для статики и изображений (Cloudflare / AWS CloudFront)
	•	Кэширование популярных запросов (Redis — v2)
3.4 Доступность и UX
	•	Адаптивный дизайн: поддержка экранов от 320px до 2560px
	•	Поддержка браузеров: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
	•	Соответствие WCAG 2.1 AA (базовая доступность)
	•	Поддержка языков: русский (основной), казахский, английский (v2)
	•	Тёмная тема — v2

5. Архитектура системы

5.1 Общая архитектура
Система построена по принципу разделения фронтенда и бэкенда (Decoupled / Headless Architecture). Фронтенд (React) и мобильное приложение взаимодействуют с бэкендом только через REST API. Это позволяет разрабатывать платформы независимо и масштабировать их раздельно.

8.2 Ключевые страницы — описание
Главная страница
	•	Hero-секция: заголовок «Найди свою стрижку», кнопка «Пройти квиз» и кнопка «Смотреть каталог»

	•	Боковая панель фильтров (десктоп) / выезжающая панель (мобайл)
	•	Активные фильтры отображаются тегами над сеткой

10.2 Договорённости по API
	•	API-контракт фиксируется в Swagger (SpringDoc OpenAPI) до начала разработки мобайла
	•	Версионирование: /api/v1/ — ничего не удалять без договорённости
	•	При изменении API — уведомить второго разработчика заранее
	•	Для тестирования использовать общую dev-среду или mocки

	•	main — стабильная продакшен-ветка, только через Pull Request

11. Глоссарий терминов

Термин
Определение
Fade
Техника стрижки: плавный переход от короткого к длинному
Taper
Похоже на fade, но менее радикальный переход
Undercut
Стрижка с выбритыми или очень короткими боками при длинном верхе
Pompadour
Стрижка с объёмным верхом, зачёсанным назад
Texture cut
Стрижка с акцентом на текстуру волос, неровные концы
Квиз
Пошаговый опросник для подбора подходящей стрижки
JWT
JSON Web Token — токен авторизации
REST API
Стандарт архитектуры веб-сервисов
DTO
Data Transfer Object — объект для передачи данных между слоями
CORS
Cross-Origin Resource Sharing — разрешение запросов с другого домена
CDN
Content Delivery Network — сеть доставки контента
MVP
Minimum Viable Product — минимально жизнеспособный продукт
