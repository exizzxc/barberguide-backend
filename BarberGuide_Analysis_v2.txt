1. Резюме проекта (Executive Summary)
BarberGuide — это B2B SaaS-платформа, соединяющая три звена: клиентов, которые не знают какую стрижку выбрать, мастеров-барберов и барбершопы/салоны. Платформа монетизируется через подписки с барбершопов и салонов — они платят за размещение, аналитику и онлайн-запись.

Ключевая идея: AI-рекомендация стрижки по форме головы и типу волос + запись к мастеру + соцсеть для работ барберов (лента в стиле TikTok/Instagram). Стартуем с MVP для валидации рынка, монетизируя с первого дня через подписки B2B.

2. Анализ рынка
2.1 Размер рынка
Рынок барберских услуг в Казахстане и СНГ активно растёт. По различным оценкам, в Казахстане насчитывается более 5 000 барбершопов и парикмахерских (только в Алматы и Астане — свыше 2 000). Средний чек в барбершопе — 3 000–8 000 тг. При подписке в 15 000–25 000 тг/мес потенциальный рынок только в Казахстане составляет 75–125 млн тг/мес при охвате 20% заведений.

2.3 Дополнительные услуги для AI-рекомендаций
Помимо стрижек, AI-модуль может рекомендовать:
	•	Оформление бороды (по форме лица и типу роста)
	•	Окрашивание волос (по тону кожи и цвету глаз)
	•	Уход за кожей головы (по типу кожи: жирная, сухая, нормальная)
	•	Укладки и текстурирующие процедуры
	•	Услуги бровей и коррекция линии роста волос
	•	Восстановительные процедуры (кератин, ботокс для волос)
Это расширяет аудиторию с мужчин до всех клиентов салонов красоты, что кратно увеличивает TAM.

3.2 Почему салоны будут платить
	•	Привлечение клиентов: пользователи, прошедшие AI-квиз, попадают в поиск барберов → живой трафик
	•	Замена Instagram: лента работ в стиле TikTok прямо на платформе — не нужно тратить время на SMM
	•	Онлайн-запись: автоматизация расписания, меньше пропущенных звонков
	•	Аналитика: какие стрижки популярны, сколько просмотров профиля, конверсия в запись
	•	QR-карточки: профессиональный инструмент общения с клиентом прямо в кресле

3.3 Стратегия выхода на рынок (GTM)
Фаза 1 — MVP-запуск (Месяц 1–2):
	•	Набрать 15–20 барбершопов вручную, предложив бесплатный период 3 мес
	•	Наполнить каталог стрижек (50–100 позиций) для демонстрации AI-квиза
	•	Запустить в 1 городе (Астана или Алматы)

Фаза 2 — Валидация (Месяц 3–4):
	•	Начать брать деньги: конвертировать первые 10 салонов на платный тариф
	•	Запустить реферальную программу: барбер приводит барбера — месяц бесплатно
	•	Собрать отзывы и доработать AI-рекомендации

Фаза 3 — Масштаб (Месяц 5+):
	•	Выход во второй город
	•	Запуск TikTok-подобной ленты работ для вирального роста
	•	Партнёрства с брендами (профессиональная косметика, инструменты)

4.2 AI-модуль: как это работает
MVP: Правиловая система (без нейросети)
	•	Пользователь проходит квиз: форма лица, тип волос, длина, стиль жизни
	•	Бэкенд матчит параметры с тегами в базе стрижек → топ-5 рекомендаций с % совпадения
	•	Быстро, дёшево, работает без ML-инфраструктуры

v2: Компьютерное зрение
	•	Пользователь загружает фото лица → CV-модель определяет форму лица автоматически
	•	Можно использовать готовые API: Face++ (бесплатно до 1000 запросов/день), Azure Face API
	•	Наложение стрижки на фото (AR-примерка) — флагманская фича для Premium

v3: Персонализация через историю
	•	ML-модель учится на предпочтениях конкретного пользователя
	•	Рекомендует новые стрижки на основе сохранённых и истории посещений

4.3 Лента работ (Social Feed) — дорожная карта
	•	v1: Портфолио барбера — галерея фото с тегами стрижек
	•	v2: Глобальная лента — все работы всех мастеров, фильтр по городу/стилю
	•	v2: Алгоритмическая лента — показывает то, что понравится конкретному пользователю
	•	v3: Видео-работы (до 60 сек), процессы стрижки, до/после
	•	v3: Реакции, сохранения, комментарии — полноценная соцсеть
Ключевое отличие от Instagram: весь контент привязан к записи. Увидел стрижку → сразу записался к этому мастеру.

8.3 Пример Go-кода: структура эндпоинта
// internal/handler/haircut_handler.go
func (h *HaircutHandler) GetHaircuts(c *gin.Context) {
    var req dto.HaircutFilterRequest
    if err := c.ShouldBindQuery(&req); err != nil {
        c.JSON(400, gin.H{"error": err.Error()})
        return
    }
    haircuts, err := h.service.GetHaircuts(c.Request.Context(), req)
    if err != nil {
        c.JSON(500, gin.H{"error": "internal server error"})
        return
    }
    c.JSON(200, haircuts)
}

Главный вывод
BarberGuide — жизнеспособный B2B-стартап с чётким болью рынка (барберы платят деньги за инструменты привлечения клиентов) и понятной монетизацией. Go-бэкенд — правильный выбор: быстрее разработка, дешевле хостинг, лучше производительность.
