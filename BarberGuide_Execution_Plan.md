# BarberGuide — Пошаговый план выполнения проекта

## Как читать этот план

План рассчитан на ~14 недель (3.5 месяца) до публичного запуска MVP, разбит на 9 фаз. Предположение: два разработчика работают в режиме, близком к full-time — ты (веб-сайт + Go-бэкенд + база данных) и партнёр (мобильное приложение на том же API). Если темп будет вечерне-выходной — умножай сроки на 1.5–2x.

Внутри каждой фазы — отдельные блоки: Backend, Frontend, и где нужно — Бизнес-трек (поиск партнёров, контент), которые двигаются параллельно с кодом, а не после него.

## ФАЗА 0 — Подготовка (Неделя 1)

Цель: к началу Недели 2 можно писать код, а не выяснять детали на ходу.

**Технический трек**
- [ ] Создать GitHub-репозиторий, ветки main / develop
- [ ] `go mod init` и установка базовых библиотек: gin, gorm, jwt, validator, migrate, swag
- [ ] Docker Compose: контейнеры go-app + postgres (Redis можно отложить)
- [ ] `.env.example` с переменными окружения (DB_DSN, JWT_SECRET, S3_KEY и т.д.)
- [ ] Зарегистрировать аккаунт на Railway/Render для будущего деплоя

**Дизайн**
- [ ] Figma-прототипы 5 ключевых экранов: главная, каталог, квиз, карточка стрижки, профиль барбера
- [ ] Зафиксировать дизайн-систему (цвета, шрифт Inter/Manrope)

**API-контракт**
- [ ] Черновик всех эндпоинтов в Swagger — даже без реализации
- [ ] Согласовать контракт с мобайл-разработчиком до того, как он начнёт верстать экраны

**Бизнес-трек**
- [ ] Таблица 20–30 барбершопов/салонов в первом городе (название, контакт, Instagram)
- [ ] One-pager: что получает салон, сколько это стоит, что бесплатно первые 3 месяца
- [ ] Назначить первые 5 звонков или встреч на Неделю 2

**Результат фазы:** окружение готово, API-контракт зафиксирован, список из 20+ потенциальных партнёров на руках.

## ФАЗА 7 — Монетизация (Неделя 13)

Цель — научиться брать деньги. Важно: не стоит интегрировать платёжный API прямо сейчас, это экономит 1–2 недели разработки и сначала проверяет готовность платить вообще.

**Подход для первых клиентов**
- [ ] Тариф хранится как поле `plan_type` в таблице `barbers` (free / basic / business / premium), переключается вручную админом после оплаты
- [ ] Оплата первых клиентов — перевод по Kaspi QR или счёт на карту, без автоматизации
- [ ] Это валидирует готовность платить до того, как тратить время на платёжный API

**Backend (минимум для MVP)**
- [ ] Ограничения функций по `plan_type` (например: free — до 5 фото в портфолио)
- [ ] Простая админ-панель или прямой доступ к БД для смены тарифа клиента
- [ ] Агрегирующие запросы для аналитики: просмотры профиля, количество записей, популярные стрижки

**Backend (после первых 10+ платящих — отдельная итерация, не сейчас)**
- [ ] Интеграция Kaspi Pay API или Stripe для автоматизации подписки

**Frontend**
- [ ] Страница тарифов (без формы оплаты — кнопка "Связаться с нами")
- [ ] Дашборд аналитики в кабинете барбера

**Результат фазы:** платформа технически различает платных и бесплатных клиентов, деньги уже поступают — пусть и вручную.

## Метрики, которые нужно отслеживать с первого дня

| Метрика | Зачем она важна |
| --- | --- |
| Кол-во зарегистрированных барберов/салонов | Базовый рост B2B-стороны |
| Конверсия free → paid | Главный показатель жизнеспособности бизнеса |
| Завершённость квиза (% дошедших до результата) | Качество UX AI-модуля |
| Конверсия "просмотр профиля барбера → запись" | Реальная ценность для барбера |
| Помесячный отток платящих салонов | Retention важнее роста на этой стадии |

## Чеклист "точка невозврата" перед публичным запуском

- [ ] Минимум 10 барбершопов реально используют платформу — не тестовые данные
- [ ] Минимум 50 стрижек в каталоге с качественными фото
- [ ] Цикл "квиз → рекомендация → запись" лично пройден 10 раз без багов
- [ ] HTTPS, бэкапы базы данных, мониторинг ошибок включены
- [ ] Согласие на обработку персональных данных и публичная оферта для платных тарифов готовы юридически
